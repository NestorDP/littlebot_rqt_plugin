# Contributing to littlebot_rqt_plugin

Thank you for your interest in contributing to littlebot_rqt_plugin! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please be respectful and inclusive in all interactions.

## How to Contribute

### Reporting Bugs

- Check existing issues to avoid duplicates
- Provide a clear description of the bug
- Include steps to reproduce
- Mention your environment (OS, ROS version, etc.)

### Suggesting Enhancements

- Clearly describe the enhancement and its benefits
- Provide use cases or examples
- Reference any relevant issues

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit with descriptive messages
6. Push to your fork
7. Create a Pull Request with a clear description

## Code Guidelines

- Follow the [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html)
- Include copyright header in all new files
- Write clear, well-documented code
- Add unit tests for new features
- Ensure code passes linting checks
